
# Etapa 5 - Testar

A etapa **Testar** valida o protótipo com usuários e coleta evidências para orientar os próximos passos. O objetivo é aprender com feedback real para reduzir riscos da solução.

## Objetivo
Verificar a usabilidade da interface e validar se o fluxo de navegação entre as telas de usuário comum, organizador e administrador está claro e intuitivo.

## Atividades da etapa
* **Sessões de teste livre:** Disponibilização do protótipo para um grupo de pessoas (familiares e amigos) para feedback sem instrução prévia.
* **Observação direta:** Acompanhamento de como os usuários interpretaram os botões de ação e se conseguiam encontrar as funções principais (como criar trilha ou denunciar).
* **Entrevistas rápidas:** Coleta de depoimentos sobre a sensação visual e facilidade de leitura.
* **Análise de fluidez:** Verificação de possíveis travamentos ou confusões no fluxo do chat e da validação de código.

## Entregáveis esperados

**Relatório de testes com principais achados:**
* **Interface Intuitiva:** O feedback geral foi muito positivo; os usuários destacaram que a interface é "limpa" e que não tiveram dificuldade em entender para que serve cada botão.
* **Navegação Fluida:** O uso do protótipo no Figma foi considerado natural. A transição entre as telas de feed e os detalhes do ponto turístico foi o ponto mais elogiado.
* **Curva de Aprendizado:** Mesmo pessoas menos familiarizadas com tecnologia conseguiram simular o envio de uma denúncia e a inscrição em uma trilha com facilidade.
* **Estética:** A escolha das cores e ícones passou uma sensação de segurança e organização, condizente com um app de trilhas.
